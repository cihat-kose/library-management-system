# Contributing Guide

Thank you for your interest in contributing.

## Workflow
1. Fork the repository.
2. Create a branch:
   - `feature/...` for new features.
   - `fix/...` for bug fixes.
3. Keep commits focused and descriptive.
4. Run local verification before opening a pull request.
5. Open a pull request that explains:
   - what changed
   - why it changed
   - how it was tested

## Code Style
- Follow existing naming and structure conventions.
- Keep user-facing messages clear and actionable.
- Update documentation when behavior changes.

## Pull Request Checklist
- [ ] Project builds successfully.
- [ ] Existing behavior is not unintentionally broken.
- [ ] Documentation is updated where needed.
